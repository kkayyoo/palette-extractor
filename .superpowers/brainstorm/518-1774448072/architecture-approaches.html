Architecture Approaches

How should we structure the application?

A

Single-Page App (Recommended)

React SPA with client-side color extraction. All processing happens in browser using Canvas API for images and a CORS proxy for CSS fetching.

Pros

  • No backend needed initially
  • Fast iteration, simpler deployment
  • Easy to add backend later
  • Free hosting (Vercel, Netlify)

Cons

  • CORS limitations for some websites
  • Heavy processing on user's device
B

Full-Stack (Next.js)

Next.js with API routes for server-side URL fetching and color extraction. More powerful but more complex.

Pros

  • No CORS issues for URL extraction
  • Server-side processing available
  • Built-in API routes
  • SEO-friendly if needed

Cons

  • More setup complexity
  • Server costs if traffic grows
  • Overkill for MVP scope
C

Hybrid (React + Serverless)

React SPA with a few serverless functions (Vercel/Netlify) for URL fetching only. Best of both worlds.

Pros

  • Client-side for most work
  • Serverless for CORS bypass
  • Scales automatically
  • Low cost, simple deployment

Cons

  • Two deployment targets
  • Slight added complexity

My recommendation: Option A (SPA) to start. We can add serverless functions for URL fetching if CORS becomes a blocker — the architecture supports this evolution without major refactoring.
