SPA vs Hybrid: Side-by-Side

Comparing the two lightweight approaches

A: Single-Page App (Pure Client)
React + Vite + Tailwind

Image Extraction

Canvas API → Extract pixels directly in browser

URL Extraction

Limited → Public CORS proxy or manual screenshot

Deployment

Static hosting (Vercel, Netlify, GitHub Pages)
Free tier sufficient
C: Hybrid (React + Serverless)
React + Vite + Tailwind + API Functions

Image Extraction

Canvas API → Same as SPA (client-side)

URL Extraction

Full Access → Serverless function fetches CSS, no CORS

Deployment

Vercel or Netlify (static + functions)
Free tier sufficient

Key Differences

Aspect A: Pure SPA C: Hybrid
Setup complexity Simpler — just React Slight extra — add /api folder
URL extraction reliability ~60% of sites (CORS blocks many) ~95% of sites (server-side fetch)
Cost $0 $0 (free tier covers typical use)
Future extensibility Can add functions later Already set up for backend growth
Dev experience Faster hot reload Nearly identical (Vite handles both)
A

Pure SPA

Simplest start. URL extraction limited but image extraction works great. Add serverless later if needed.

C

Hybrid (Recommended)

Minimal extra setup for reliable URL extraction. Since URL input is a core feature, worth doing right from the start.

Updated recommendation: Given that URL extraction is a core feature (not just nice-to-have), Option C makes more sense. The serverless function is ~20 lines of code and ensures the URL feature actually works reliably.
